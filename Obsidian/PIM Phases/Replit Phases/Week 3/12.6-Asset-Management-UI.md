# Phase 12.6: Asset Management UI

**Execution Plan Item:** #14 (Asset management portion)  
**Replit Phase:** 12.6 (Week 3 follow-on; depends on Phase 12.3 Asset Manager)  
**Dependencies:** Asset Manager (Phase 12.3), UI Shell (Phase 07), shared components from Product UI (Phase 12) where applicable (ChannelLocalePicker, AttributeValueInput including MediaFileInput)  
**Depended on by:** Product edit page can offer asset picker for `assetSingle` and `assetList` attributes (optional enhancement). Asset type admin (defining types and assigning attributes) is Phase 12.7.

---

## Objective

Build the **Asset Management UI** so users can create, list, and edit assets from the PIM. The pattern mirrors Entity and Product UI: list with filters, create flow, edit page with attribute values (including file upload for media attributes) and channel/locale context — no variants or categories. **Per Phase 12.1:** Where locale/channel fallback applies, show first available fallback value with an inheritance/fallback indicator; allow editing (override) on the current asset.

---

## 1. Sidebar & Navigation

- Add an **Assets** section to the sidebar (e.g., under **Products** or **Entities**). "All Assets" (or "Assets") links to `/assets`.
- Filter by asset type on the list page; optional expandable types in sidebar.

---

## 2. Asset List Page (`/assets`)

### Layout

- **PageHeader:** "Assets" title, breadcrumbs, "+ Create" button.
- **Search bar:** Search by asset code and/or label/metadata.
- **Filter bar:** Asset type dropdown (All Types + each type).
- **Count:** "Showing N assets."
- **Table columns:** Code (link to edit), Type (asset type label), optional thumbnail (from first mediaFile attribute if present), Updated (relative time).
- **Pagination:** Cursor-based, page size selector.
- **Row click:** Navigate to `/assets/:code`.
- **Empty state:** "No assets yet. Create your first asset." with Create button.

### Create Button

Opens create flow: select asset type, then code (or auto-generate). Create → redirect to edit page.

---

## 3. Asset Create Flow

- Modal or two-step: (1) Choose asset type. (2) Code (required, camelCase, unique).
- **Mutation:** `POST /api/v1/assets` with `assetTypeCode` and `code`. On success, invalidate list cache and navigate to `/assets/:code`.

---

## 4. Asset Edit Page (`/assets/:code`)

### Data Loading

- Parallel queries: asset (GET `/api/v1/assets/:code`), values (GET `/api/v1/assets/:code/values`), asset type with attributes (GET `/api/v1/asset-types/:code`), locales, channels. Same TanStack Query pattern as Product/Entity edit.

### Layout

- **Breadcrumbs:** "Assets / ASSET-CODE".
- **Title:** Asset code (and optional label from attribute if configured).
- **Context panel (left):** Channel/Locale picker for scoped attributes.
- **Main content:** Attribute values grouped by attribute group; each field via **AttributeValueInput**. Ensure **MediaFileInput** is used for `mediaFile` attributes (upload, preview, remove). Other types (text, date, single select, etc.) same as Product/Entity edit.

### Save Behavior

- Explicit Save button; dirty state; only changed values to `PUT /api/v1/assets/:code/values`. Spinner, success toast, inline validation errors and scroll to first error. Optional unsaved-changes confirmation on navigate away.

---

## 5. Shared Components

- Reuse **ChannelLocalePicker**, **AttributeValueInput** (including mediaFile), attribute group collapsible sections, validation/error patterns from Phase 12.
- **Asset-specific:** Asset type badge on list and edit; thumbnail column on list (from media attribute if available); create form (type + code).

---

## 6. Routes

- `GET /assets` → Asset list page.
- `GET /assets/:code` → Asset edit page.
- Register in the same router as products and entities.

---

## 7. Exit Criteria

- [ ] Users can open the Assets list, filter by type, search, and open an asset edit page.
- [ ] Users can create a new asset (choose type, set code) and are redirected to edit.
- [ ] Users can edit attribute values (including uploading/replacing media files) per channel/locale and save; only changed values sent; errors shown inline.
- [ ] Product edit page can reference assets by code in `assetSingle` and `assetList` attributes; optional: asset picker (e.g., dropdown or modal) that lists assets of a relevant type.
