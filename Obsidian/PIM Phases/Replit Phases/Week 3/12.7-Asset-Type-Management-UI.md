# Phase 12.7: Asset Type Management UI

**Execution Plan Item:** #14 (Asset management — structure/config portion)  
**Replit Phase:** 12.7 (Week 3 follow-on; depends on Phase 12.3 Asset Manager)  
**Dependencies:** Asset Manager (Phase 12.3), UI Shell (Phase 07), shared components from Task 7 where applicable (PageHeader, form patterns, LocaleLabelsEditor for labels, DataTable/list patterns)  
**Depended on by:** None (completes admin for asset types; assets and asset edit already use types from Phase 12.6)

---

## Objective

Build the **Asset Type Management UI** so users can create, list, and edit asset types and assign or remove attributes to each type. The pattern mirrors the Families admin (Task 7): list page with search/pagination, create flow, and edit page with basic info (code, labels) and assigned-attributes section with add/remove/reorder. Asset types are flat "templates" for assets — no variants, no requirements table in MVP. All backend APIs exist in Phase 12.3; this phase is UI only.

---

## 1. Sidebar & Navigation

- Add **Asset Types** to the **Structure** section of the sidebar (alongside Families, Attributes, Attribute Groups, Categories).
- Use an appropriate icon (e.g. `Layers` or `FolderKanban` from lucide-react to distinguish from "Assets" which uses `Image`).
- **Label:** "Asset Types".
- **Link:** `/asset-types`.
- Active state: highlight when route is `/asset-types` or `/asset-types/:code`.

## 2. Asset Types List Page (`/asset-types`)

### Layout

- **PageHeader:** "Asset Types" title, breadcrumbs, "+ Create" (or "Create asset type") button.
- **Search bar:** Optional. Search by asset type code (and optionally label) via query param on `GET /api/v1/asset-types?search=&limit=&cursor=`.
- **Count:** "Showing N asset types" with correct singular/plural.
- **Table columns:**

| Column    | Description |
|-----------|-------------|
| Code      | camelCase code, font-medium, clickable (navigate to edit). |
| Label     | Localized label in user's locale (fallback to code if missing). |
| Attributes| Count of assigned attributes (from API `attributeCount` if returned). |
| Updated   | Last updated, relative time (e.g. date-fns `formatDistanceToNow`). |

- **Pagination:** Cursor-based (Previous/Next); optional page size selector (e.g. 25 default).
- **Row click:** Navigate to `/asset-types/:code`.
- **Empty state:** "No asset types yet. Create your first asset type to get started." with Create button.
- **Loading:** Spinner or skeleton while data loads.

### Data Fetching

- `GET /api/v1/asset-types?limit=N&cursor=X&search=Y` via TanStack Query.
- Query key pattern: e.g. `["/api/v1/asset-types", "?{params}"]` for cache and invalidation consistency with other list pages.

## 3. Asset Type Create Flow

**Recommended: Modal dialog** (matches simple-entity pattern; keeps navigation on list).

- **Component:** `AssetTypeCreateDialog` (or similar). Triggered by the "+ Create" button on the list page.
- **Form fields:**
  - **Code** — Required. Text input validated against camelCase regex (`^[a-z][a-zA-Z0-9]*$`). Server-side uniqueness on submit.
  - **Labels** — Optional for MVP, or multi-locale label editor (reuse `LocaleLabelsEditor` from Task 7 if available). API accepts `labels` as JSONB `{ "en_US": "Product Photo", ... }`.
- **Validation:** react-hook-form + Zod; inline errors per field.
- **Mutation:** `POST /api/v1/asset-types` with body `{ code, labels }`.
- **On success:** Invalidate asset-types list query, success toast, close dialog, optionally navigate to `/asset-types/:code` so the user can add attributes.
- **On error:** Destructive toast with message; keep form state.
- **Cancel:** Close dialog without saving.

**Alternative: Full-page create.** Route `/asset-types/new`, form with Code + Labels, submit → `POST` → redirect to `/asset-types/:code`. Use if the app prefers full-page creates for structure entities (e.g. like Families).

## 4. Asset Type Edit Page (`/asset-types/:code`)

### Data Loading

- `GET /api/v1/asset-types/:code` — asset type with assigned attributes array (each with `attributeCode`, `sortOrder` or equivalent).
- For "Add attribute" picker: `GET /api/v1/attributes?limit=500` (and optionally `GET /api/v1/attribute-groups?limit=500`) to show attributes not yet on the type, optionally grouped.

### Layout

- **Breadcrumbs:** "Asset Types / {code}" with "Asset Types" linking back to `/asset-types`.
- **Title:** Asset type code (and optional localized label).

### Section 1: Basic info

- **Code** — Read-only on edit (immutable per Phase 12.3).
- **Labels** — Editable. Multi-locale label editor (one field per active locale or reuse shared component). Save via `PATCH /api/v1/asset-types/:code` with `{ labels }` only.
- **Save** button for this section, or single Save for entire page if preferred.

### Section 2: Assigned attributes

- **List** of attributes currently assigned to this asset type. Columns: Attribute code, Attribute label (localized), Type (badge), Sort order. Optional: Remove button per row.
- **Reorder:** Drag-and-drop or up/down buttons to change order. Persist via re-add with new sort order, or a dedicated reorder API if added later. Phase 12.3 supports `sortOrder` in `POST .../attributes`; reorder can be implemented by removing and re-adding with new order, or by a future `PATCH` on the junction if available.
- **Add attribute:** "Add attribute" button opens a picker (dropdown or modal) listing attributes that are not already assigned. Filter to attributes not in the current list. On select: `POST /api/v1/asset-types/:code/attributes` with body `{ attributes: [{ attributeCode: "..." }] }` or `{ attributeCode, sortOrder }`. Then refetch asset type or invalidate query.
- **Remove attribute:** Per-row "Remove" (or trash icon). Confirmation: "Remove {attributeCode} from this asset type? Assets of this type will no longer have this attribute." Then `DELETE /api/v1/asset-types/:code/attributes/:attributeCode`. Refetch/invalidate.

### Delete asset type

- **Delete** button (e.g. in header or danger zone). Confirmation: "Delete asset type {code}? This will fail if any assets of this type exist." On confirm: `DELETE /api/v1/asset-types/:code`. If 400 (assets exist), show error toast with message. On success: invalidate list, redirect to `/asset-types`, success toast.

### Error handling

- **Not found:** If `GET /api/v1/asset-types/:code` returns 404, redirect to `/asset-types` with toast "Asset type not found."
- **Validation errors:** Inline on form; toast for API errors.

## 7. Backend API (no changes)

All endpoints are defined in Phase 12.3. This phase consumes them only.

| Method | Endpoint | Use in Phase 12.7 |
|--------|----------|-------------------|
| GET | `/api/v1/asset-types?limit=&cursor=&search=` | List page. |
| GET | `/api/v1/asset-types/:code` | Edit page (type + attributes). |
| POST | `/api/v1/asset-types` | Create asset type. |
| PATCH | `/api/v1/asset-types/:code` | Update labels. |
| DELETE | `/api/v1/asset-types/:code` | Delete type (with guard: 400 if assets exist). |
| POST | `/api/v1/asset-types/:code/attributes` | Add attribute(s) to type. |
| DELETE | `/api/v1/asset-types/:code/attributes/:attributeCode` | Remove attribute from type. |
