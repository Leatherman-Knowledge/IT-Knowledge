# Phase 12.8: Entity Type Management UI

**Execution Plan Item:** #11 (Entities — structure/config portion)  
**Replit Phase:** 12.8 (Week 3 follow-on; depends on Phase 12.2 Entities)  
**Dependencies:** Entities (Phase 12.2), UI Shell (Phase 07), shared components from Task 7 where applicable (PageHeader, form patterns, LocaleLabelsEditor for labels, DataTable/list patterns)  
**Depended on by:** None (completes admin for entity types; entities and entity edit already use types from Phase 12.5)

---

## Objective

Build the **Entity Type Management UI** so users can create, list, and edit entity types and assign or remove attributes to each type. The pattern mirrors the Asset Type Management UI (Phase 12.7) and Families admin (Task 7): list page with search/pagination, create flow, and edit page with basic info (code, labels) and assigned-attributes section with add/remove/reorder. Entity types are flat "templates" for entities — no variants. Requirements per channel/locale (entity_type_requirements) are deferred in 12.2; omit from this UI unless a future phase adds them. All backend APIs exist in Phase 12.2; this phase is UI only.

## 2. Entity Types List Page (`/entity-types`)

### Layout

- **PageHeader:** "Entity Types" title, breadcrumbs, "+ Create" (or "Create entity type") button.
- **Search bar:** Optional. Search by entity type code (and optionally label) via query param on `GET /api/v1/entity-types?search=&limit=&cursor=`.
- **Count:** "Showing N entity types" with correct singular/plural.
- **Table columns:**

| Column    | Description |
|-----------|-------------|
| Code      | camelCase code, font-medium, clickable (navigate to edit). |
| Label     | Localized label in user's locale (fallback to code if missing). |
| Attributes| Count of assigned attributes (from API `attributeCount` if returned). |
| Updated   | Last updated, relative time (e.g. date-fns `formatDistanceToNow`). |

- **Pagination:** Cursor-based (Previous/Next); optional page size selector (e.g. 25 default).
- **Row click:** Navigate to `/entity-types/:code`.
- **Empty state:** "No entity types yet. Create your first entity type to get started." with Create button.
- **Loading:** Spinner or skeleton while data loads.

### Data Fetching

- `GET /api/v1/entity-types?limit=N&cursor=X&search=Y` via TanStack Query.
- Query key pattern: e.g. `["/api/v1/entity-types", "?{params}"]` for cache and invalidation consistency with other list pages.

## 3. Entity Type Create Flow

**Recommended: Modal dialog** (matches Asset Type Management and simple-entity pattern).

- **Component:** `EntityTypeCreateDialog` (or similar). Triggered by the "+ Create" button on the list page.
- **Form fields:**
  - **Code** — Required. Text input validated against camelCase regex (`^[a-z][a-zA-Z0-9]*$`). Server-side uniqueness on submit.
  - **Labels** — Optional for MVP, or multi-locale label editor (reuse `LocaleLabelsEditor` from Task 7 if available). API accepts `labels` as JSONB `{ "en_US": "Brand", ... }`.
- **Validation:** react-hook-form + Zod; inline errors per field.
- **Mutation:** `POST /api/v1/entity-types` with body `{ code, labels }`.
- **On success:** Invalidate entity-types list query, success toast, close dialog, optionally navigate to `/entity-types/:code` so the user can add attributes.
- **On error:** Destructive toast with message; keep form state.
- **Cancel:** Close dialog without saving.

**Alternative: Full-page create.** Route `/entity-types/new`, form with Code + Labels, submit → `POST` → redirect to `/entity-types/:code`. Use if the app prefers full-page creates for structure entities.

## 4. Entity Type Edit Page (`/entity-types/:code`)

### Data Loading

- `GET /api/v1/entity-types/:code` — entity type with assigned attributes array (each with `attributeCode`, `sortOrder` or equivalent).
- For "Add attribute" picker: `GET /api/v1/attributes?limit=500` (and optionally `GET /api/v1/attribute-groups?limit=500`) to show attributes not yet on the type, optionally grouped.

### Layout

- **Breadcrumbs:** "Entity Types / {code}" with "Entity Types" linking back to `/entity-types`.
- **Title:** Entity type code (and optional localized label).

### Section 1: Basic info

- **Code** — Read-only on edit (immutable per Phase 12.2).
- **Labels** — Editable. Multi-locale label editor (one field per active locale or reuse shared component). Save via `PATCH /api/v1/entity-types/:code` with `{ labels }` only.
- **Save** button for this section, or single Save for entire page if preferred.

### Section 2: Assigned attributes

- **List** of attributes currently assigned to this entity type. Columns: Attribute code, Attribute label (localized), Type (badge), Sort order. Optional: Remove button per row.
- **Reorder:** Drag-and-drop or up/down buttons to change order. Persist via re-add with new sort order, or a dedicated reorder API if added later. Phase 12.2 supports `sortOrder` in `POST .../attributes`; reorder can be implemented by removing and re-adding with new order, or by a future `PATCH` on the junction if available.
- **Add attribute:** "Add attribute" button opens a picker (dropdown or modal) listing attributes that are not already assigned. Filter to attributes not in the current list. On select: `POST /api/v1/entity-types/:code/attributes` with body `{ attributes: [{ attributeCode: "..." }] }` or `{ attributeCode, sortOrder }`. Then refetch entity type or invalidate query.
- **Remove attribute:** Per-row "Remove" (or trash icon). Confirmation: "Remove {attributeCode} from this entity type? Entities of this type will no longer have this attribute." Then `DELETE /api/v1/entity-types/:code/attributes/:attributeCode`. Refetch/invalidate.

### Delete entity type

- **Delete** button (e.g. in header or danger zone). Confirmation: "Delete entity type {code}? This will fail if any entities of this type exist." On confirm: `DELETE /api/v1/entity-types/:code`. If 400 (entities exist), show error toast with message. On success: invalidate list, redirect to `/entity-types`, success toast.

### Error handling

- **Not found:** If `GET /api/v1/entity-types/:code` returns 404, redirect to `/entity-types` with toast "Entity type not found."
- **Validation errors:** Inline on form; toast for API errors.

## 7. Backend API (no changes)

All endpoints are defined in Phase 12.2. This phase consumes them only.

| Method | Endpoint | Use in Phase 12.8 |
|--------|----------|-------------------|
| GET | `/api/v1/entity-types?limit=&cursor=&search=` | List page. |
| GET | `/api/v1/entity-types/:code` | Edit page (type + attributes). |
| POST | `/api/v1/entity-types` | Create entity type. |
| PATCH | `/api/v1/entity-types/:code` | Update labels. |
| DELETE | `/api/v1/entity-types/:code` | Delete type (with guard: 400 if entities exist). |
| POST | `/api/v1/entity-types/:code/attributes` | Add attribute(s) to type. |
| DELETE | `/api/v1/entity-types/:code/attributes/:attributeCode` | Remove attribute from type. |

## 8. Exit Criteria

- [ ] Users can open the Entity Types list, search (optional), and open an entity type edit page.
- [ ] Users can create a new entity type (code, optional labels) and are taken to the edit page or back to list.
- [ ] Users can edit entity type labels and save via PATCH.
- [ ] Users can add attributes to an entity type and remove attributes from an entity type; sort order is visible and can be updated (reorder) if supported by API or workflow.
- [ ] Users can delete an entity type when no entities of that type exist; appropriate error is shown when entities exist.
