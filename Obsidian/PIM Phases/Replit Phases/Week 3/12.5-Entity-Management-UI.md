# Phase 12.5: Entity Management UI

**Execution Plan Item:** #14 (Entity management portion)  
**Replit Phase:** 12.5 (Week 3 follow-on; depends on Phase 12.2 Entities)  
**Dependencies:** Entities (Phase 12.2), UI Shell (Phase 07), shared components from Product UI (Phase 12) where applicable (e.g., ChannelLocalePicker, AttributeValueInput)  
**Depended on by:** Product edit page can offer entity picker for `entityReference` attributes (optional enhancement). Entity type admin (defining types and assigning attributes) is Phase 12.8.

---

## Objective

Build the **Entity Management UI** so users can create, list, and edit reference entities (metaobjects) from the PIM. The interaction pattern mirrors the Product UI where applicable: list with filters, create flow, edit page with attribute values and channel/locale context — but without variants or categories. **Per Phase 12.1:** Where locale/channel fallback applies, show first available fallback value with an inheritance/fallback indicator; allow editing (override) on the current entity.

---

## 1. Sidebar & Navigation

- Add an **Entities** section to the sidebar (e.g., under **Products** or under **Structure**). Placement can match execution plan preference (e.g., "Entity and asset management screens following the same interaction patterns as products").
- **All Entities** (or "Entities") links to `/entities`.
- Optional: type filter in sidebar (e.g., expandable list of entity types) or keep a single "All Entities" and filter by type on the list page.

---

## 2. Entity List Page (`/entities`)

### Layout

- **PageHeader:** "Entities" title, breadcrumbs, "+ Create" button.
- **Search bar:** Search by entity code and/or by label attribute value (if entity type has a label attribute).
- **Filter bar:** Entity type dropdown (All Types + each entity type). Optional: channel/locale filters if list shows scoped summary.
- **Count:** "Showing N entities."
- **Table columns:** Code (link to edit), Type (entity type label), optional label/value column, Updated (relative time). Optional: status if entities get a status field later.
- **Pagination:** Cursor-based, page size selector (e.g., 25 default).
- **Row click:** Navigate to `/entities/:code`.
- **Empty state:** "No entities yet. Create your first entity." with Create button.

### Create Button

Opens a create flow: select entity type, then optionally enter code (or auto-generate). Redirect to edit page after create.

---

## 3. Entity Create Flow

- **Option A (single step):** Modal or inline: Entity type dropdown (required), Code (required, camelCase). Create → redirect to `/entities/:code`.
- **Option B (two steps):** Step 1 — Choose entity type. Step 2 — Code (optional if auto-generated). Create → redirect to edit page.
- Validation: entity type required; code camelCase and unique. Use react-hook-form + zod if following Product create pattern.
- **Mutation:** `POST /api/v1/entities` with `entityTypeCode` and `code`. On success, invalidate list cache and navigate to edit.

---

## 4. Entity Edit Page (`/entities/:code`)

### Data Loading

- Parallel queries: entity (GET `/api/v1/entities/:code`), values (GET `/api/v1/entities/:code/values`), entity type with attributes (GET `/api/v1/entity-types/:code`), locales, channels (for picker). Reuse TanStack Query patterns from Product edit.

### Layout

- **Breadcrumbs:** e.g., "Entities / ENTITY-CODE".
- **Title:** Entity code (and optional label from attribute if configured).
- **Context panel (left):** Channel/Locale picker (reuse ChannelLocalePicker). Entities can have localizable/channelizable attributes; channel list can be all channels or derived from usage (MVP: all channels).
- **Main content:** Attribute values grouped by attribute group (same grouping and collapsible sections as Product edit). Each field rendered via shared **AttributeValueInput** (Phase 12) where possible.

### Attribute Scoping

- Same rules as Product edit: global always shown; localizable when locale selected; channelizable when channel selected; both when both selected. Read-only axis attributes do not apply (no variants).

### Save Behavior

- Explicit Save button; dirty state tracking; only changed values sent to `PUT /api/v1/entities/:code/values`. Spinner on save; toast on success; inline errors and scroll to first error on validation failure. Optional: "Unsaved changes" confirmation on navigate away.

---

## 5. Shared Components

- Reuse **ChannelLocalePicker**, **AttributeValueInput** (all 15 types), attribute group collapsible sections, and validation/error display patterns from Phase 12.
- **Entity-specific:** Entity type badge or label on list and edit; entity create form (type + code). No need for CompletenessIndicator unless entity types get requirements and completeness is implemented later.

---

## 6. Routes

- `GET /entities` → Entity list page.
- `GET /entities/:code` → Entity edit page.
- Register in the same router as products (e.g., under AuthenticatedRouter).

---

## 7. Exit Criteria

- [ ] Users can open the Entities list, filter by type, search, and open an entity edit page.
- [ ] Users can create a new entity (choose type, set code) and are redirected to edit.
- [ ] Users can edit attribute values per channel/locale and save; only changed values are sent; errors shown inline.
- [ ] Product edit page can reference entities by code in `entityReference` attributes; optional: entity picker dropdown (Phase 12 or this phase) that lists entities of a type relevant to the attribute.
